2022-6-26

ui组件库打包的特点是保持它的原目录层级；
使用babel api 直接打包；
使用less、sass 分别打包样式文件；
拷贝其他类型文件；
如何解决 每个 pkg 内都进行打包；

为什么有 lib es 三个目录 fusion/next 的源码中；

2022-6-30

其实前端最难的就是 编译！

如果强项在于搞编译，也是很牛逼了；

然后以编译散开：基建、devopts 等等、npm部署；

任何跨端、uniapp 等等，到最后都是通过编译！

所以后期还是有一个专项，如何进行 编译；

这次的ui库，编译的问题算解决了，后期再研究；

然后就是研究 文档的问题；
